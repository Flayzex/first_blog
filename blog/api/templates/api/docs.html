Документация API "articles"

GET /api/articles/

Получение списка всех статей.

Параметры запроса:

Нет.

Ответ:

  • Код состояния: 200 OK
  • Тело ответа: список статей в формате JSON.

POST /api/articles/

Создание новой статьи.

Параметры запроса:

  • title (обязательный): заголовок статьи.
  • content (обязательный): содержание статьи.

Ответ:

  • Код состояния: 201 Created
  • Тело ответа: созданная статья в формате JSON.

GET /api/articles/<int:pk>/

Получение деталей определенной статьи по ее идентификатору.

Параметры запроса:

  • pk (обязательный): идентификатор статьи.

Ответ:

  • Код состояния: 200 OK
  • Тело ответа: детали статьи в формате JSON.

PUT /api/articles/<int:pk>/

Обновление определенной статьи по ее идентификатору.

Параметры запроса:

  • pk (обязательный): идентификатор статьи.
  • title (опциональный): новый заголовок статьи.
  • content (опциональный): новое содержание статьи.

Ответ:

  • Код состояния: 200 OK
  • Тело ответа: обновленная статья в формате JSON.

DELETE /api/articles/<int:pk>/

Удаление определенной статьи по ее идентификатору.

Параметры запроса:

  • pk (обязательный): идентификатор статьи.

Ответ:

  • Код состояния: 204 No Content

GET /api/articles/delete/<int:pk>/

(Устаревший эндпоинт) Получение деталей определенной статьи по ее идентификатору.

Параметры запроса:

  • pk (обязательный): идентификатор статьи.

Ответ:

  • Код состояния: 200 OK
  • Тело ответа: детали статьи в формате JSON.

Авторизация и разрешения

Для доступа к эндпоинтам API требуется аутентификация пользователя. Каждый запрос должен содержать заголовок Authorization, в котором передается логин и пароль доступа.

Разрешения

  • IsOwnerOrReadOnly: Это разрешение позволяет только владельцу статьи выполнять операции обновления и удаления.

Заключение

Это документация для API "articles", которая описывает доступные эндпоинты, параметры запросов, разрешения и сериализацию данных. Убедитесь, что вы настроили аутентификацию и разрешения в соответствии с вашими потребностями.
